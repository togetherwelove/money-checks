# 공유 가계부 기획 및 설계 문서

## 1. 개요

본 문서는 알뜰의 공유 가계부 기능에서 발생하는 공유코드 발급, 참여 요청, 소유자 승인/거절, 멤버 방출, 탈퇴, 플랜 제한, 다운그레이드 제한 정책을 정의한다.

공유 방식은 `공유코드로 참여 요청 → 소유자 승인` 구조로 확정한다. 소유자가 특정 사용자를 직접 초대하고 대상자가 수락하는 별도 초대 시스템은 지원하지 않는다.

## 2. 플랜 정책

### 2.1 Free 플랜

| 항목 | 제한 |
| --- | --- |
| 최대 접근 가능 가계부 수 | 1개 |
| 포함 범위 | 직접 생성한 가계부 + 공유코드로 승인받아 참여 중인 가계부 |
| 공유 인원 수 | 본인 포함 최대 2명 |
| 공유 가능 여부 | 가능 |
| 다중 가계부 참여 | 불가 |

### 2.2 Plus 플랜

| 항목 | 제한 |
| --- | --- |
| 최대 접근 가능 가계부 수 | 3개 |
| 포함 범위 | 직접 생성한 가계부 + 공유코드로 승인받아 참여 중인 가계부 |
| 공유 인원 수 | 본인 포함 최대 5명 |
| 공유 가능 여부 | 가능 |
| 다중 가계부 참여 | 가능 |

### 2.3 제한 계산 기준

가계부 수 제한은 다음을 모두 합산한다.

- 사용자가 직접 생성한 개인 가계부
- 사용자가 직접 생성한 공유 가계부
- 다른 사용자의 가계부에 참여 중인 공유 가계부

| 사용자 플랜 | 직접 생성 | 참여 중 | 추가 참여 가능 여부 |
| --- | ---: | ---: | --- |
| Free | 1개 | 0개 | 원칙적으로 불가. 단, 기존 개인 가계부 삭제 처리 안내 후 참여 요청 가능 |
| Free | 0개 | 1개 | 불가 |
| Plus | 2개 | 1개 | 불가 |
| Plus | 1개 | 1개 | 가능, 1개 남음 |

## 3. 핵심 용어

| 용어 | 설명 |
| --- | --- |
| 가계부 | 수입, 지출, 날짜 메모, 카테고리 등을 관리하는 기본 단위 |
| 개인 가계부 | 소유자 1명만 사용하는 가계부 |
| 공유 가계부 | 2명 이상의 사용자가 함께 사용하는 가계부 |
| 소유자 | 가계부를 생성한 사용자 |
| 편집자 | 공유 가계부에 참여하여 조회/등록/수정/삭제 권한을 가진 사용자 |
| 공유코드 | 공유 가계부 참여 요청을 보내기 위한 16자리 코드 |
| 참여 요청 | 사용자가 공유코드를 입력하여 소유자에게 참여 승인을 요청하는 행위 |
| 방출 | 소유자가 특정 편집자를 공유 가계부에서 제거하는 행위 |
| 탈퇴 | 편집자가 스스로 공유 가계부에서 나가는 행위 |

## 4. 권한 체계

### 4.1 권한 종류

| 권한 | 설명 |
| --- | --- |
| 소유자 | 공유코드 확인/공유, 참여 요청 승인/거절, 편집자 방출, 가계부 이름 변경, 조건 충족 시 가계부 삭제 가능 |
| 편집자 | 수입/지출/날짜 메모/카테고리 조회, 등록, 수정, 삭제 가능 |

관리자 권한은 별도로 두지 않는다. 멤버 역할은 `소유자 / 편집자`만 유지한다.

### 4.2 소유자 제한

소유자는 공유 가계부에서 나갈 수 없다. 소유권 이전 기능은 지원하지 않는다.

소유자의 가계부 삭제는 편집자가 아무도 없는 경우에만 허용한다. 편집자가 남아 있는 공유 가계부는 먼저 모든 편집자를 방출한 뒤 삭제할 수 있다.

## 5. 공유코드 정책

### 5.1 공유코드 형식

공유코드는 16자리 코드만 지원한다. 앱 링크 또는 딥링크 초대는 지원하지 않는다.

### 5.2 공유코드 유효기간

공유코드 유효기간은 7일로 한다.

유효기간이 지난 공유코드는 사용할 수 없으며, 사용자는 소유자에게 새 공유코드를 요청해야 한다.

## 6. 참여 요청 정책

### 6.1 기본 흐름

1. 소유자가 공유코드를 확인하고 필요한 사용자에게 전달한다.
2. 참여하려는 사용자가 공유코드를 입력한다.
3. 서버가 공유코드 유효성, 공유 인원 제한, 요청자 가계부 수 제한을 검증한다.
4. 검증을 통과하면 참여 요청이 생성된다.
5. 소유자에게 참여 요청이 표시되고 알림을 보낼 수 있다.
6. 소유자가 승인 또는 거절한다.
7. 승인 시 요청자는 편집자로 공유 가계부에 참여한다.

### 6.2 참여 요청 상태

| 상태 | 설명 |
| --- | --- |
| pending | 소유자 승인 대기 중 |
| approved | 소유자 승인 완료 |
| rejected | 소유자 거절 |

요청자가 직접 참여 요청을 취소하는 기능은 제공하지 않는다.

### 6.3 참여 요청 유효기간

참여 요청 유효기간은 7일로 한다. 유효기간이 지난 요청은 승인할 수 없도록 처리한다.

### 6.4 거절 후 재요청

소유자가 참여 요청을 거절한 경우 같은 사용자는 다시 요청할 수 있다. 단, 내부적으로 20초 정도의 짧은 쿨다운을 두어 반복 요청을 방지한다.

## 7. 참여 승인 시 검증 조건

참여 승인 시점에는 서버에서 다음 조건을 다시 검증한다.

- 참여 요청이 존재하는가?
- 참여 요청 상태가 pending인가?
- 참여 요청 유효기간이 지나지 않았는가?
- 공유 가계부의 현재 인원이 소유자 플랜의 공유 인원 제한을 초과하지 않는가?
- 요청자의 현재 접근 가능 가계부 수가 요청자 플랜 제한을 초과하지 않는가?
- 요청자가 Free 플랜이고 개인 가계부 1개를 이미 가진 경우, 기존 개인 가계부 삭제 처리 대상인지 확인 가능한가?

## 8. Free 사용자의 기존 개인 가계부 처리

Free 사용자가 개인 가계부 1개를 가진 상태에서 공유 가계부에 참여하려는 경우, 앱은 참여 요청 전 기존 개인 가계부가 삭제된다는 점을 강하게 안내한다.

별도의 세부 선택지는 제공하지 않는다. 참여 요청을 계속 진행하면 다음 정책에 동의한 것으로 본다.

- 소유자가 요청을 승인하면 기존 개인 가계부는 더 이상 사용하지 않는다.
- 기존 개인 가계부 전체가 삭제된다.
- 삭제 범위에는 가계부, 수입/지출 내역, 날짜 메모, 첨부파일 등 기존 개인 가계부에 속한 데이터 전체가 포함된다.
- 병합은 지원하지 않는다.

소유자가 승인하기 전까지는 기존 개인 가계부를 계속 사용할 수 있다. 실제 삭제는 승인 시점에 처리한다.

## 9. 공유 가계부 전환 및 데이터 접근

개인 가계부에 다른 사용자가 참여하면 해당 가계부는 실질적으로 공유 가계부가 된다.

공유 가계부에 참여한 사용자는 참여 전 작성된 기존 내역 전체를 조회할 수 있다. 이는 공유 가계부의 기본 정책으로 확정한다.

## 10. 방출 정책

### 10.1 방출 조건

소유자만 편집자를 방출할 수 있다.

방출 대상은 소유자가 아니어야 한다. 소유자는 방출할 수 없다.

### 10.2 방출 후 처리

방출된 사용자는 해당 가계부 접근 권한을 즉시 잃는다.

방출된 사용자가 작성한 기존 내역은 공유 가계부에 남긴다.

작성자명은 유지하되, 방출되었거나 더 이상 접근 권한이 없는 사용자는 희미한 빨간색 텍스트의 작성자명으로 구분한다.

방출된 사용자에게 알림을 보낼 수 있다.

## 11. 탈퇴 정책

편집자는 공유 가계부에서 스스로 탈퇴할 수 있다.

탈퇴 후 처리 정책은 방출과 동일하다.

- 가계부 접근 권한 즉시 제거
- 과거 작성 내역 유지
- 작성자명 유지
- 더 이상 접근 권한이 없는 사용자는 희미한 빨간색 텍스트로 구분

소유자는 공유 가계부에서 나갈 수 없다. 소유권 이전은 지원하지 않는다.

## 12. 플랜 변경 정책

### 12.1 Free에서 Plus로 업그레이드

업그레이드 후 즉시 다음 제한이 확장된다.

| 항목 | 변경 전 | 변경 후 |
| --- | ---: | ---: |
| 최대 접근 가능 가계부 수 | 1개 | 3개 |
| 공유 인원 | 2명 | 5명 |

가능해지는 기능:

- 추가 가계부 생성
- 추가 공유 가계부 참여 요청
- 공유 인원 추가 승인

### 12.2 Plus에서 Free로 다운그레이드

다운그레이드 즉시 기존 데이터를 삭제하지 않는다. 대신 제한 초과 상태로 전환한다.

제한 초과 상태에서는 다음 기능을 제한한다.

- 새 가계부 생성 불가
- 새 공유 가계부 참여 요청 불가
- 새 참여 요청 승인 불가
- 등록/수정/삭제 불가
- 조회만 가능

공유 인원이 Free 제한인 2명을 초과한 경우에도 기존 참여자를 자동 방출하지 않는다. 해당 가계부는 제한 초과 상태가 되며, 소유자는 인원을 2명 이하로 줄이거나 Plus로 재가입해야 정상 편집할 수 있다.
