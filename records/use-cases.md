# 유스케이스 기록

최종 갱신: 2026-04-03
상태: 적용 완료

## 목적

테스트를 작성하기 전에 사용자 관점의 요구사항과 예외 흐름을 먼저 고정한다.

앞으로 기능을 추가하거나 수정할 때는 아래 순서를 기본으로 사용한다.

1. 유스케이스 작성 또는 수정
2. 테스트 작성
3. 구현
4. 리팩토링

## UC-01 로그인

### 목표

사용자가 Google 계정으로 앱에 로그인한다.

### 성공 흐름

1. 사용자가 로그인 화면에서 Google 로그인을 누른다.
2. 계정을 선택한다.
3. 앱 세션이 생성된다.
4. 별명 입력 화면이 열린다.
5. 사용자가 이름을 입력하고 저장한다.
6. 알림 권한 화면이 열린다.
7. 사용자가 기기 권한 팝업에서 허용 여부를 선택한다.
8. 달력 화면으로 진입한다.

### 예외 흐름

- OAuth 실패 시 로그인 화면에 남는다.
- 세션 초기화에 실패하면 로그인 화면으로 돌아가고 오류 메시지를 보여준다.
- 로그인 취소 시 세션은 생성되지 않는다.
- 계정 선택 화면에서 뒤로 가면 로그인 화면에 남고, 다시 로그인하면 계정 선택 화면이 다시 열린다.
- 프로필 조회가 지연되거나 실패해도 별명 입력 또는 메인 진입이 무한 로딩으로 막히지 않는다.
- 이름 저장 실패 시 다음 단계로 넘어가지 않는다.
- 알림 권한을 거부해도 달력 화면으로는 진입할 수 있다.

## UC-02 내역 수동 입력

### 목표

사용자가 수입 또는 지출 내역을 직접 입력해 저장한다.

### 성공 흐름

1. 사용자가 기록 화면으로 이동한다.
2. 날짜 영역을 누르고 캘린더 또는 시스템 날짜 선택기에서 날짜를 고른다.
3. 유형, 금액, 분류, 메모를 입력한다.
4. 저장 버튼을 누른다.
5. 내역이 현재 가계부에 저장된다.
6. 달력과 목록에 반영된다.

### 예외 흐름

- 사용자는 헤더의 달력 이동 버튼으로 저장 없이 달력 화면으로 돌아갈 수 있다.
- 필수 입력이 부족하면 저장되지 않는다.
- 저장 실패 시 오류 메시지를 보여준다.

## UC-03 내역 수정

### 목표

사용자가 기존 내역을 수정한다.

### 성공 흐름

1. 사용자가 기존 내역을 선택한다.
2. 필요하면 날짜 영역을 누르고 캘린더 또는 시스템 날짜 선택기에서 날짜를 바꾼다.
3. 수정 화면에서 값을 변경한다.
4. 저장한다.
5. 변경된 내역이 목록과 달력에 반영된다.

### 예외 흐름

- 사용자는 헤더의 달력 이동 버튼으로 저장 없이 달력 화면으로 돌아갈 수 있다.
- 수정 권한이 없으면 반영되지 않는다.
- 저장 실패 시 기존 값은 유지된다.

## UC-04 내역 삭제

### 목표

사용자가 기존 내역을 삭제한다.

### 성공 흐름

1. 사용자가 내역 삭제를 누른다.
2. 내역이 현재 가계부에서 삭제된다.
3. 목록과 달력에서 제거된다.

### 예외 흐름

- 삭제 권한이 없으면 삭제되지 않는다.
- 삭제 실패 시 목록은 유지된다.

## UC-05 공유 가계부 참여 요청

### 목표

사용자가 공유 코드를 입력해 참여 요청을 보낸다.

### 성공 흐름

1. 사용자가 공유 가계부 화면에서 코드를 입력한다.
2. 참여 요청 보내기를 누른다.
3. 요청이 `pending` 상태로 저장된다.
4. 소유자에게 참여 요청 알림이 간다.

### 예외 흐름

- 코드가 잘못되면 오류 메시지를 보여준다.
- 만료된 코드면 만료 오류를 보여준다.
- 재요청 제한 시간 안이면 대기 오류를 보여준다.
- 사용자가 이미 다른 공유 가계부의 소유자 또는 편집자면 요청을 막는다.

## UC-06 공유 가계부 참여 승인

### 목표

소유자가 참여 요청을 승인한다.

### 성공 흐름

1. 소유자가 참여 요청 목록을 연다.
2. 승인 버튼을 누른다.
3. 요청자가 멤버로 추가된다.
4. 요청 상태가 `approved`로 변경된다.
5. 요청자의 활성 가계부가 해당 가계부로 바뀐다.

### 예외 흐름

- 소유자가 아니면 승인할 수 없다.
- 이미 처리된 요청이면 승인할 수 없다.

## UC-07 공유 가계부 멤버 강퇴

### 목표

소유자가 특정 멤버를 공유 가계부에서 제외한다.

### 성공 흐름

1. 소유자가 멤버 목록에서 강퇴를 누른다.
2. 대상 사용자가 해당 가계부에서 제거된다.
3. 대상 사용자는 개인 가계부로 돌아간다.
4. 관련 알림이 발생한다.

### 예외 흐름

- 소유자가 아니면 강퇴할 수 없다.
- 존재하지 않는 멤버는 제거되지 않는다.

## UC-08 앱 메뉴 열기

### 목표

사용자가 헤더의 메뉴 버튼으로 현재 화면 위에 메뉴 드로어를 연다.

### 성공 흐름

1. 사용자가 헤더 우측 상단의 메뉴 버튼을 누른다.
2. 메뉴 드로어가 오른쪽에서 왼쪽으로 부드럽게 열린다.
3. 사용자가 원하는 메뉴 항목을 선택한다.
4. 현재 화면에서 해당 화면으로 이동하고 메뉴 드로어는 닫힌다.

### 예외 흐름

- 바깥 영역을 누르면 메뉴 드로어만 닫히고 현재 화면은 유지된다.
- 알림설정 기능이 비활성화된 상태면 해당 메뉴 항목은 보이지 않는다.

## UC-09 알림 설정 변경

### 목표

사용자가 알림 설정 화면에서 알림 옵션을 변경한다.

### 성공 흐름

1. 사용자가 메뉴에서 알림설정으로 이동한다.
2. 금액 기준 또는 공유 활동 알림 옵션을 변경한다.
3. 설정이 로컬 저장소에 반영된다.

### 예외 흐름

- 기기 알림 권한이 거부된 경우 알림 자체는 오지 않는다.
- 필수 알림은 화면에 노출되지 않으며 항상 켜진 상태로 유지된다.

## UC-10 계정 삭제

### 목표

사용자가 자신의 계정과 관련 데이터를 완전히 삭제한다.

### 성공 흐름

1. 사용자가 계정 화면 하단의 계정 삭제 영역을 연다.
2. 확인 문구 `삭제`를 입력한다.
3. 계정 삭제 버튼을 누른다.
4. 서버 함수가 현재 사용자를 확인한다.
5. `auth.users` 삭제가 실행된다.
6. 연결된 프로필, 가계부 기록, 참여 정보가 FK cascade로 함께 삭제된다.
7. 로컬 세션이 종료된다.

### 예외 흐름

- 다른 멤버가 있는 공유 가계부 소유자면 삭제를 막는다.
- 인증 토큰이 없으면 삭제를 진행하지 않는다.
- 서버 함수 실패 시 삭제되지 않는다.

## UC-11 영수증 OCR 초안 작성

### 목표

사용자가 영수증 이미지를 업로드해 가계부 초안을 자동 작성한다.

### 현재 상태

계획 단계다. 구현 전까지는 아래 흐름을 기준으로 테스트와 설계를 준비한다.

### 성공 흐름

1. 사용자가 기록 화면에서 영수증 스캔을 선택한다.
2. 영수증 이미지를 촬영하거나 선택한다.
3. OCR 서비스가 한국어 영수증 텍스트를 추출한다.
4. 총액, 날짜, 상호명 등이 초안으로 채워진다.
5. 사용자가 수정 후 저장한다.

### 예외 흐름

- OCR 실패 시 직접 입력으로 전환한다.
- 일부 필드만 추출되면 추출된 값만 초안에 반영한다.

## UC-12 월간 분석 확인

### 목표

사용자가 달력 화면에서 이번 달 수입/지출 추이와 카테고리별 지출 비중을 확인한다.

### 성공 흐름

1. 사용자가 달력 화면으로 이동한다.
2. 사용자가 달력 화면에서 차트 화면으로 이동한다.
3. 현재 선택된 월의 총수입과 총지출을 본다.
4. 전월 대비 수입과 지출이 얼마나 늘었는지 또는 줄었는지 확인한다.
5. 카테고리별 지출 비중을 원형 차트와 목록으로 확인한다.
6. 월을 바꾸면 같은 분석이 해당 월 기준으로 갱신된다.

### 예외 흐름

- 해당 월 지출이 없으면 카테고리 차트 대신 빈 상태를 보여준다.
- 전월 데이터가 없으면 비교 기준은 0원으로 처리한다.
