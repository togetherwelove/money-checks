# 배지 알림 정책

## 상태

- 정책 결정일: 2026-07-23
- 상태: 확정

## 목적

앱 아이콘 배지와 화면 내 빨간 점을 동일한 미확인 상태에서 계산한다.
사용자가 관련 화면을 확인하면 배지가 즉시 제거되고, 이미 확인한 알림이 앱 재실행이나
알림 센터 동기화로 다시 미확인 상태가 되지 않아야 한다.

## 배지 의미

- 배지는 처리되지 않은 작업이 아니라 아직 확인하지 않은 새 소식을 의미한다.
- 내역 추가 알림은 `전체 내역` 화면을 열면 확인한 것으로 처리한다.
- 참여 요청 알림은 `공유` 화면을 열면 확인한 것으로 처리한다.
- 참여 요청이 승인 또는 거절되지 않았더라도 화면에서 확인했다면 배지는 제거한다.
- 관련 화면을 보고 있는 동안 도착한 새 알림은 즉시 확인한 것으로 처리한다.

## 배지 대상

- 다른 구성원이 추가한 입출금 내역
- 소유자에게 도착한 가계부 참여 요청
- 참여 요청 읽음 처리는 가계부 소유자에게만 적용하며, 편집자가 `공유` 화면을 열어도
  참여 요청 읽음 RPC를 호출하지 않는다.

금액 초과, 구성원 가입·탈퇴·방출 등 나머지 푸시 알림에는 앱 아이콘 배지와 화면 내
배지를 표시하지 않는다.

## 여러 가계부

- 읽음 처리는 현재 선택된 가계부에만 적용한다.
- 다른 가계부에 미확인 내역이 있으면 `전체 내역` 푸터 배지는 유지한다.
- 가계부 전환 목록에서 가계부별 미확인 배지를 표시해 사용자가 남은 위치를 찾을 수 있게 한다.
- `공유` 푸터 배지는 참여 요청이 있는 가계부 중 아직 확인하지 않은 요청이 하나라도 있으면 표시한다.

## 저장 및 동기화

- Supabase를 읽음 상태의 기준 저장소로 사용한다.
- 사용자, 가계부, 배지 종류별 마지막 확인 시각을 저장한다.
- 내역과 참여 요청의 서버 생성 시각이 마지막 확인 시각보다 새로우면 미확인으로 계산한다.
- 기존 사용자에게 과거 전체가 미확인으로 표시되지 않도록 마이그레이션 시각을 최초 확인 기준으로 저장한다.
- 다른 기기에서 읽음 처리한 결과는 앱이 활성 상태일 때 실시간으로 반영한다.
- 백그라운드 기기의 앱 아이콘 배지는 다음 앱 활성화 시 서버 상태로 교정한다.

## 앱 아이콘 및 알림 센터

- 앱 아이콘 숫자는 모든 가계부의 미확인 내역 수와 미확인 참여 요청 수의 합계다.
- 화면 내 푸터는 숫자 대신 빨간 점으로 표시한다.
- 관련 화면에서 읽음 처리할 때 해당 가계부와 종류에 해당하는 알림 센터 항목도 제거한다.
- 앱 실행 시 알림 센터 항목이나 마지막 알림 응답을 읽음 상태의 원본으로 사용하지 않는다.
- 처리한 마지막 알림 응답은 삭제하여 다음 앱 실행에서 다시 처리하지 않는다.

## 실패 및 예외 처리

- 서버 읽음 처리에 실패하면 화면 배지와 앱 아이콘 상태를 성공한 것으로 먼저 변경하지 않는다.
- 클라이언트는 현재 사용자의 가계부 소유 여부를 먼저 확인하고, 서버에서 허용하지
  않는 참여 요청 읽음 처리를 시도하지 않는다.
- 서버 배지 상태 조회에 실패하면 마지막으로 성공한 서버 상태를 로컬 캐시에서 사용한다.
- 마지막 성공 상태도 없으면 오래된 앱 아이콘 배지를 제거하고 빈 상태로 재시도한다.
- 권한 또는 런처 제한으로 앱 아이콘 배지 변경이 실패해도 화면 내 배지 상태는 서버 기준을 유지한다.
