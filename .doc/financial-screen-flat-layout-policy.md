# 금액 화면 평면형 레이아웃 정책

## 상태

- 정책 결정일: 2026-07-23
- 상태: 확정

## 목적

가계부 금액을 확인하거나 입력하는 핵심 화면에서 미색 배경과 카드가 반복되는 단계감을
제거한다. 흰 배경 위에 여백과 얇은 구분선을 사용해 정보를 단순하고 빠르게 읽을 수
있도록 한다.

## 적용 범위

- 홈
- 전체 내역
- 통계
- 내역 등록 및 수정

인증, 계정, 앱 설정, 구독, 후원 화면은 이번 변경 범위에서 제외한다.

## 공통 원칙

- 핵심 금액 화면의 콘텐츠 배경은 순백색을 사용한다.
- 구조를 만들기 위한 외곽 카드의 배경색, 테두리, 둥근 모서리는 제거한다.
- 화면 너비를 크게 차지하는 구조적 컨테이너에는 둥근 모서리를 사용하지 않는다.
- 버튼, 필터 칩, 날짜 셀처럼 상태와 조작 대상을 직접 표현하는 작은 요소의 둥근
  모서리는 유지한다.
- 섹션 구분은 여백과 얇은 구분선을 우선 사용한다.
- 카드 제거 후에도 기존 정보 순서와 기능은 변경하지 않는다.
- 공통 헤더와 푸터의 기존 디자인은 유지한다.
- 광고 배경과 구분선은 콘텐츠 좌우 패딩에 제한되지 않고 화면 전체 너비까지
  이어지게 한다.
- 배너 및 네이티브 광고 영역의 배경은 순백색으로 통일한다.
- 광고 소재 자체의 크기와 가운데 정렬은 광고 SDK의 기존 동작을 유지한다.

## 화면별 정책

### 홈

- 달력, 월간 요약, 선택 날짜 내역을 같은 흰 배경 위에 배치한다.
- 월간 요약의 별도 배경은 제거하고 위아래 구분선은 유지한다.
- 광고 영역의 시각적 구분은 유지한다.
- pull-to-refresh는 선택 날짜 내역 목록에만 적용하되, 내역이 적거나 없어도 목록의
  남은 빈 공간에서 시작할 수 있어야 한다.

### 전체 내역

- 화면과 내역 행 배경을 흰색으로 통일한다.
- 내역 행은 구분선으로만 나눈다.
- 필터 칩은 선택 상태를 전달하는 상호작용 요소이므로 유지한다.
- 전체 내역 리스트는 화면 전체 너비를 사용하고, 일반 내역 행에만 콘텐츠 좌우
  패딩을 적용해 네이티브 광고가 잘리지 않게 한다.
- 네이티브 광고의 배경과 구분선은 전체 너비를 유지하되, 아이콘과 문구 등 내부
  콘텐츠는 일반 내역과 같은 좌우 패딩으로 정렬한다.

### 통계

- 도넛 차트와 막대 차트의 외곽 카드 배경, 테두리, 둥근 모서리를 제거한다.
- 배너 광고는 월별 추이와 원그래프보다 앞선 차트 화면 최상단에 배치한다.
- 통계 섹션은 여백과 구분선으로 나눈다.
- 카테고리별 및 구성원별 분할 선택기는 적용 대상 그래프 바로 앞에 배치한다.
- 사용자는 선택한 뒤 변경된 그래프 제목과 데이터를 순서대로 확인한다.
- 원그래프 아래의 별도 항목 활성화 영역은 제거한다.
- 원그래프 우측 목록에는 활성 및 비활성 항목을 모두 표시하고, 각 행을 눌러 그래프
  포함 여부를 전환한다.
- 우측 목록은 기존 원그래프 높이 안에서 스크롤되며, 모든 항목을 제외한 상태에서도
  다시 항목을 활성화할 수 있어야 한다.
- 우측 목록의 마지막에는 다음 항목이 약 절반 노출되도록 행 높이를 조정해 스크롤
  가능한 영역임을 시각적으로 전달한다.
- 차트, 범례, 선택 상태, 수입 및 지출 강조색은 유지한다.

### 내역 등록 및 수정

- 입력 폼 전체를 감싸는 외곽 카드를 제거한다.
- 금액, 내용, 메모 입력란은 배경색, 외곽선, 둥근 모서리를 제거하고 밑줄로 입력
  영역을 표시한다.
- 지출 및 수입 선택기는 외곽 배경과 테두리 없이 하단 기준선과 선택 항목의 상태색
  밑줄로 표시한다.
- 지출 및 수입 선택기는 상단 날짜 영역에서 금액 입력란 바로 위로 이동하고, `금액`
  라벨을 대신하는 제목형 선택기로 사용한다.
- 분류 선택 그리드는 휴대폰 화면 한 줄에 최소 6개 항목이 배치되도록 셀 크기를
  반응형으로 계산한다.
- 입력 항목 사이는 기존 여백 구조를 유지한다.

## 유지 대상

- 버튼, 필터, 모달, 광고
- 달력 선택 상태와 히트맵
- 수입 및 지출 상태색
- 로딩, 오류, 빈 상태
