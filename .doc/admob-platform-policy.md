# AdMob 플랫폼 설정 정책

## 상태

- 정책 결정일: 2026-07-23
- 상태: 확정

## 앱 ID

- Android와 iOS의 AdMob 앱 ID를 Expo 네이티브 설정에 각각 등록한다.
- 앱 ID는 `ca-app-pub-숫자~숫자` 형식이어야 한다.
- 앱 ID가 없거나 형식이 잘못되면 네이티브 빌드 전에 설정 단계에서 실패시킨다.
- 앱 ID와 광고 단위 ID는 환경 변수로 관리한다.
- 동적 Expo 설정을 평가하기 전에 Expo 표준 환경 로더로 로컬 `.env`를 읽는다.
- EAS가 주입한 환경 변수는 로컬 파일 값으로 덮어쓰지 않는다.

## 광고 단위 ID

- 배너, 네이티브, 전면, 보상형 전면 광고는 실행 플랫폼에 맞는 광고 단위 ID를 사용한다.
- 개발 빌드는 Google Mobile Ads 라이브러리가 제공하는 플랫폼별 테스트 ID를 사용한다.
- 비개발 빌드는 EAS 빌드 프로필에 등록된 실제 광고 단위 ID를 사용한다.
- 지원하지 않는 플랫폼이거나 실제 광고 단위 ID가 없으면 광고를 렌더링하지 않는다.

## 기존 노출 정책

- 광고는 기존 정책대로 Free 사용자에게만 노출한다.
- Plus 사용자의 기능 흐름은 광고 없이 동일하게 제공한다.
- 네이티브 광고는 현재의 아이콘, 제목, 본문, 행동 버튼 구성을 유지한다.

## 빌드

- 네이티브 앱 ID 변경은 OTA 업데이트로 반영할 수 없으므로 새 Android 또는 iOS 빌드가 필요하다.
